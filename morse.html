Morse Code Learning Device

BME 60A Capstone · Team of 4 · Arduino / C++ · Fall 2025

An interactive Arduino device that teaches Morse code and translates in both directions — press the button in real time and it decodes dots and dashes into text on the serial monitor; type a message and it plays it back as Morse on a buzzer with synchronized LED feedback. We framed it as an accessibility tool, inspired by research on Morse-based communication for people with cerebral palsy and speech or motor impairments.

▶ Device demo — live encode/decode and two-player messaging
My Contribution Timing & input

I owned the button-press timing and input sensitivity — the logic that turns how long a button is held into a dot or a dash, and a potentiometer that lets the user tune the press thresholds to their own motor ability (important for the accessibility use case).

  • Timing logic — used millis() to measure press duration and classify dot vs. dash against tunable dot/dash thresholds, with debouncing to prevent spurious inputs.
  • Adjustable sensitivity — a potentiometer sets the timing window so users with different motor control can calibrate the device to themselves.
  • Two-player mode — extended the input handling so two people can exchange messages, each with their own button.
Stages Built
  • Read — button presses decode into dots/dashes on the serial monitor with buzzer and red-LED feedback.
  • Transcribe — a 48-entry Morse lookup (letters, numbers, punctuation) converts patterns into characters.
  • Communicate — two-player messaging, plus a serial-to-Morse mode that plays a typed message back audibly.
